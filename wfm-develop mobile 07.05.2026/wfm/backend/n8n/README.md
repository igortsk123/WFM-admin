# n8n — конвейеры автоматизации WFM

Папка содержит JSON-экспорты конвейеров (workflows) для [n8n](https://n8n.io/), которые автоматизируют рутинные операции в системе WFM.

## Назначение

n8n используется для задач, которые не являются частью основной бизнес-логики сервисов, но необходимы для работы системы:

- **Периодические операции** — cron-задачи: закрытие смен, генерация планов, очистка данных
- **Эксперименты и прототипы** — быстрая проверка гипотез без деплоя кода
- **Интеграции** — взаимодействие с внешними системами (LAMA, уведомления и т.д.)
- **Обслуживание данных** — миграции, исправления, массовые операции

## Конвейеры

| Файл | Расписание | Описание |
|------|-----------|----------|
| `lama_daily_sync.json` | Ежедневно 6:00 | Полная синхронизация из LAMA: сотрудники → смены → задачи по всем магазинам (PROD) |
| `lama_daily_sync_dev.json` | Ежедневно 6:00 | То же + очистка `external_id` у всех задач — тестовые данные не улетают обратно в LAMA (DEV) |
| `daily_shift_and_task.json` | Ежедневно 0:00 | Тестовый конвейер: создаёт смену и задачу для John Doe (sso_id=409809c4…) |
| `Создание плановых смен.json` | Каждый час | Создаёт записи в `shifts_plan` на текущий день для заданных пользователей |
| `Вечернее закрытие смен.json` | Ежедневно 23:50 | Закрывает все открытые фактические смены (`shifts_fact`) |
| `pg_admin.json` | — | Утилита для работы с PostgreSQL |
| `semetrics_references_csv.json` | По запросу (webhook) | Отдаёт справочники для Semantic Layer Semetrics: `users.csv`, `stores.csv`, `assignments.csv`. См. `.memory_bank/backend/guides/semetrics_references.md` |

## Как использовать

1. Открыть n8n:
   - **DEV**: `https://dev.wfm.beyondviolet.com/`
   - **PROD**: `https://wfm.beyondviolet.com/`
2. Импортировать JSON-файл через меню **Workflows → Import from File**
3. Настроить credentials для PostgreSQL (подключение к БД `shifts`)
4. Активировать конвейер

## Подключение к БД

SQL-конвейеры работают напрямую с PostgreSQL. Credentials настраиваются в n8n и ссылаются по ID:

| Credential ID | БД | Назначение |
|---|---|---|
| `tasks` | `wfm_tasks` | Задачи и смены (shifts переехали сюда) |
| `users` | `wfm_users` | Пользователи и назначения |

`lama_daily_sync.json` работает через HTTP: делает `POST http://svc_tasks:8000/internal/sync-daily` — не требует прямого доступа к БД.

## Важно

- JSON-файлы содержат **ID credentials**, но не сами пароли — безопасно хранить в git
- При импорте на новый инстанс n8n нужно заново привязать credentials
- Перед активацией конвейера на production — проверить SQL-запросы вручную
