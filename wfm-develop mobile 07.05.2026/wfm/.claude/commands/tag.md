# tag

Автоматически определить версию тега, создать и запушить его.

## Алгоритм

1. Получи последние теги: `git tag --sort=-creatordate | head -20`
2. Получи коммиты с момента последнего тега только на ветке develop: `git log <last_tag>..origin/develop --oneline --no-merges`
   - Если список пуст — сообщи пользователю, что новых коммитов нет, и не создавай тег
3. Определи тип бампа по префиксам коммитов:
   - **MAJOR** (`BREAKING`, `feat!`, `fix!`) → увеличь первую цифру, сбрось остальные
   - **MINOR** (`feat(`) → увеличь вторую цифру, сбрось третью
   - **PATCH** (всё остальное: `fix(`, `chore(`, `refactor(`, `docs(` и т.д.) → увеличь третью цифру
4. Покажи пользователю: текущий тег, новый тег, список коммитов — и спроси подтверждение
5. После подтверждения: `git tag <new_tag> && git push origin <new_tag>`

## Формат версии

Теги вида `vMAJOR.MINOR.PATCH` (например `v0.1.10` → следующий патч `v0.1.11`).
