# Спецификация устройства

## Guition / Jingcai JC4827W543N

### 4.3" ESP32-S3 HMI Display Module без сенсорной панели

---

## 1. Общее описание

**JC4827W543N** — встраиваемый HMI/LCD-модуль на базе **ESP32-S3-WROOM-1 / ESP32-S3-N4R8** с цветным **4.3-дюймовым TFT IPS-дисплеем 480×272**, беспроводными интерфейсами **Wi-Fi** и **Bluetooth**, встроенным слотом **microSD / TF**, поддержкой питания от литиевой батареи и возможностью разработки в Arduino IDE, ESP IDE, MicroPython и Guition. Версия **N** обозначает исполнение **NTP / Without Touch Panel**, то есть без сенсорной панели. ([Surenoo Display][1])

## 9. Распиновка microSD / TF-слота

Встроенный слот **microSD / TF** подключён по **SPI**. Для него лучше явно объявлять отдельный SPI-интерфейс, а не полагаться на пины по умолчанию.

| microSD / TF  |       GPIO |
| ------------- | ---------: |
| SD CS         | **GPIO10** |
| SD MOSI       | **GPIO11** |
| SD SCK / SCLK | **GPIO12** |
| SD MISO       | **GPIO13** |

В рабочем решении для JC4827W543 microSD инициализируется через отдельный `SPIClass spiSD{ HSPI }`, с пинами `SD_MISO 13`, `SD_MOSI 11`, `SD_SCLK 12`, `SD_CS 10`, далее вызывается `spiSD.begin(SD_SCLK, SD_MISO, SD_MOSI, SD_CS)` и `SD.begin(SD_CS, spiSD)`. ([GitHub][2])

Рекомендуемая формулировка для спецификации:

> **microSD / TF card interface:** встроенный слот microSD подключён по SPI. Используемые линии: **CS — GPIO10, MOSI — GPIO11, SCK — GPIO12, MISO — GPIO13**. Для стабильной работы необходимо явно инициализировать SPI-интерфейс для SD-карты и не полагаться на SPI-пины по умолчанию.

Пример инициализации для Arduino:

```cpp
#include <SPI.h>
#include <SD.h>

#define SD_CS    10
#define SD_MOSI  11
#define SD_SCK   12
#define SD_MISO  13

SPIClass spiSD(HSPI);

void setup() {
  Serial.begin(115200);

  spiSD.begin(SD_SCK, SD_MISO, SD_MOSI, SD_CS);

  if (!SD.begin(SD_CS, spiSD)) {
    Serial.println("microSD mount failed");
    return;
  }

  Serial.println("microSD mounted");
}
```
