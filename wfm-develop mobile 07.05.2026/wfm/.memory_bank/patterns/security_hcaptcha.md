# hCaptcha — защита от ботов

## Что это

**hCaptcha** — система защиты от автоматизированных атак и ботов, используемая в процессе авторизации и регистрации.

### Преимущества

- Приватность пользователей (не отслеживает, в отличие от reCAPTCHA)
- Работает в России и странах с ограничениями
- Бесплатный план до 1M запросов/месяц
- Поддержка всех платформ: Web, iOS, Android

---

## Когда показывается

hCaptcha может потребоваться на любом этапе авторизации/регистрации:

1. **При запросе кода** (отправка номера телефона)
2. **При регистрации** (отправка данных нового пользователя)
3. **При подтверждении кода** (в редких случаях)

**Решение принимает сервер** на основе:
- Частоты запросов с IP
- Подозрительного поведения
- Истории блокировок номера

Клиент должен быть готов показать капчу при получении ошибки `AUTH_CAPTCHA_REQUIRED`.

---

## Как это работает

### 1. Запрос без капчи

Клиент отправляет обычный запрос (например, запрос кода):

```
POST /oauth/authorize/
phone=+79998887777
notification_type=phone_code
```

### 2. Сервер требует капчу

Если нужна защита, сервер возвращает:

```json
{
  "status": {
    "code": "AUTH_CAPTCHA_REQUIRED",
    "message": "Для продолжения требуется ввод captcha"
  }
}
```

### 3. Пользователь решает капчу

Клиент показывает hCaptcha widget. После успешного решения получает токен.

### 4. Повторный запрос с токеном

Клиент повторяет **тот же запрос**, добавив параметр `captcha`:

```
POST /oauth/authorize/
phone=+79998887777
notification_type=phone_code
captcha=P0_eyJ0eXAiOiJKV1QiLCJhbGc...
```

### 5. Сервер проверяет и продолжает

Сервер верифицирует токен и продолжает обычную логику.

---

## Ключевые особенности

### Токены одноразовые

Каждый токен hCaptcha можно использовать **только один раз**. Повторное использование приведет к ошибке.

### Время жизни токена

Токен действителен **120 секунд** после решения капчи. После истечения нужно показать капчу заново.

### Неинвазивная интеграция

hCaptcha добавляется **без изменения** существующего API — просто как опциональный параметр `captcha` в запросах.

---

## Конфигурация

### Site Key (публичный ключ)

Используется в клиентских приложениях (Web, iOS, Android):

```
18e5142d-9054-487b-af5d-ce24cf8a09f9
```

### Secret Key (приватный ключ)

Хранится **только на backend** в переменных окружения. Используется для верификации токенов через API hCaptcha.

---

## Обработка ошибок

### AUTH_CAPTCHA_REQUIRED

Показать hCaptcha widget пользователю.

### AUTH_CAPTCHA_INVALID

Токен неверный или истек. Показать капчу заново.

### Сетевые ошибки при загрузке капчи

Показать сообщение "Проверьте подключение к интернету" и кнопку "Повторить".

---

## Альтернативные способы авторизации

Если пользователь не может решить капчу:
- Предложить получить код через звонок (вместо SMS/Telegram)
- Показать инструкцию по решению капчи
- Предоставить контакт поддержки

---

## Итого

hCaptcha защищает систему от ботов, при этом:
- Не усложняет обычный flow для честных пользователей
- Показывается только при подозрительной активности
- Интегрируется прозрачно через параметр `captcha`
- Работает на всех платформах одинаково

Подробности интеграции для каждой платформы см. в соответствующих stack документах (web/, mobile/, backend/).
