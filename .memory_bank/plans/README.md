# Активные планы

Эта папка содержит планы, которые находятся в работе.

## Шаблон плана

При создании нового плана используй этот шаблон:

```markdown
# План: {Название}

**Статус:** В работе
**Создан:** {YYYY-MM-DD}
**Последнее обновление:** {YYYY-MM-DD}

---

## Цель

{Краткое описание цели плана}

## Задачи

### {Раздел 1}

- [ ] Задача 1.1
- [ ] Задача 1.2

### {Раздел 2}

- [ ] Задача 2.1
- [ ] Задача 2.2

---

## Лог выполнения

### {YYYY-MM-DD}

- Начало работы над планом
```

## Правила

1. **Галочки**: `[ ]` — не выполнено, `[x]` — выполнено
2. **При выполнении задачи**: добавь дату и комментарий если нужно
   ```
   - [x] Задача 1.1 — 2026-01-21, реализовано
   ```
3. **Лог**: веди записи о значимых действиях и проблемах
4. **Обновление документации**: в конце каждого плана **обязательно** добавляй задачи по обновлению `.memory_bank/` — это нужно почти всегда:
   - обновить описание экранов / API / доменных моделей если они изменились
   - добавить новые паттерны или архитектурные решения
   - обновить ссылки в `.memory_bank/README.md` если добавлены новые файлы документации
   - затем обновить ссылки в `CLAUDE.md`
5. **Завершение**: перемести файл в `completed_plans/` когда все задачи выполнены
