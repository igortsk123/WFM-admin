# MY-PLAN-3: Reports cleanup в Tasks Review + Task Detail

**Размер:** средний.
**Зависимости:** ничего критичного (можно после MY-PLAN-2).
**Затрагивает:** Tasks Review + Task Detail + Reject Dialog.

## Контекст бизнес-логики (от user)

Реалистичный отчёт сотрудника о выполнении задачи:
- Сотрудник **никак не отчитывается текстом** в обычной задаче — это нереалистично
- Максимум — отмечает галочками выполненные подзадачи (если у задачи есть подзадачи)
- Photo может быть/не быть — определяется в настройках конкретной задачи (`requires_photo: boolean`)
- Если `requires_photo === false` — нет смысла показывать пустой блок «Фото обязательно, но не приложено»

## Задачи

### 3.1. Tasks Review (`/tasks/review`) — отчёт detail panel

**Файл:** `components/features/tasks/review-queue.tsx` (или где рендерится detail).

Изменения в правой колонке "Отчёт исполнителя":
- **Убрать** большой text report block (не показывать поле "Текстовый отчёт от сотрудника")
- **Если у задачи есть подзадачи** (subtasks?.length > 0):
  - Показать **"Выполненные подзадачи"** — list checkbox (read-only) с текстом подзадачи и галочкой (`completed: true/false`)
  - Title секции переименовать: "Отчёт исполнителя" → "Выполненные подзадачи"
- **Если подзадач нет** — секцию отчёта **скрывать целиком** (нечего показывать)
- **Photo секция:**
  - Если `task.requires_photo === false` И `report_image_url === null` → секцию **не показывать**
  - Если `requires_photo === true` И фото нет → показать предупреждение "Фото обязательно, но не приложено" (это валидно — система должна была не дать сабмитнуть)
  - Если фото есть → показать фото (через `<Image>` если есть размеры)

### 3.2. Task Detail (`/tasks/[id]`) — Tab "Отчёт"

**Файл:** `components/features/tasks/task-detail.tsx`

Текущий Tab "report" содержит:
- Текстовый отчёт (textarea с `report_text`)
- Фото (image preview)
- Время выполнения

Изменения:
- **Убрать** textarea "Текстовый отчёт"
- Photo block: same logic как в 3.1
- Время выполнения OK (оставить)

### 3.3. Task Detail (`/tasks/[id]`) — Tab "Подзадачи"

**Файл:** тот же.

Текущее: Tab "subtasks" с возможностью **частично подтвердить** (checkbox по каждой подзадаче).

Изменения:
- **Убрать** частичное подтверждение (checkbox approve per-subtask убираем)
- Только **просмотр** статуса каждой подзадачи (✓ выполнена / ☐ не выполнена)
- Добавить **кнопку "Предложить отредактировать"** на каждой подзадаче — для роли STORE_DIRECTOR (управляющий магазина):
  - Click → Dialog с textarea "Предложите новый текст подзадачи"
  - Submit → создаёт subtask suggestion с source='store_director' → попадает в `/subtasks/moderation`
  - После — toast «Предложение отправлено супервайзеру»
- Для остальных ролей кнопки "Предложить отредактировать" disabled с tooltip «Только управляющий магазина может предлагать»

### 3.4. Task Detail — Reject flow

**Файл:** `components/features/tasks/reject-dialog-content.tsx`

Текущий: textarea с reject reason + опциональный category.

Изменения:
- Если у задачи есть **невыполненные подзадачи** (incomplete) → автозаполнить textarea: `НЕСДЕЛАНО: <название первой невыполненной подзадачи>`
  - User может перезаписать или дописать
- Checkbox "Добавить НЕСДЕЛАНО к комментарию задачи" — если выставлен, при reject ещё и комментарий задачи обновляется автоматически

### 3.5. Reject reasons enum cleanup

**Файлы:** `messages/*.json` + reject-dialog-content компонент.

Текущие reasons:
- "Не релевантно сейчас" (для AI suggestions, не для tasks reject)
- "Уже в работе"
- "Не согласен с анализом"
- "Слишком общее"
- "Другое"

Это больше для AI suggestions. Для **task reject** нужны другие:
- ❌ Убрать "Нет фото" (если оно было — система не даст в принципе сабмитнуть без фото если требуется)
- ✅ "Фото не по теме"
- ✅ "Работа не выполнена"
- ✅ **Добавить:** "Срок просрочен / выполнено поздно"
- ✅ Свободный текст — оставить textarea для деталей

Если есть отдельный enum `screen.tasksReview.reject_reasons` — обновить там. Если шарится с AI — разделить.

### 3.6. Сортировка в Tasks Review по умолчанию

**Файл:** `components/features/tasks/review-queue.tsx`

Сейчас сортировка может быть произвольная. По умолчанию должны быть **новые сверху** (самые свежие на проверку). Найти sort-state initial → установить `created_at desc` или `submitted_at desc` (что-то время-ориентированное).
