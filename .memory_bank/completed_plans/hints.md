**Статус:** Выполнено

**Последнее обновление:** 2026-03-05

## Цель

Реализовать первый этап раздела «Подсказки» на экране деталей задачи (iOS/Android).
Статичные подсказки — фиксированный набор из до 6 текстовых советов, привязанных
к конкретному типу операции и зоне. Данные из файла
`Список_задач_с_подсказками_уникальные_зоны_v2_3.xlsx`.

Второй этап (ИИ-чат) — отдельный сервис, в рамках этого плана не реализуется.

---

## Архитектурное решение

### Проблема
В `operation_types` сейчас 7 широких категорий (Выкладка, Переоценка, …),
а Excel работает с **62 специфическими шаблонами задач** (TaskID 1–62:
проверка цен, ротация, фейсинг…). Подсказки должны быть привязаны к конкретному
шаблону, а не к широкой категории.

### Решение
- Добавить **62 новые записи** в таблицу `operation_types` (IDs 8–69), соответствующие
  специфическим шаблонам из Excel. Существующие 7 записей остаются без изменений.
- Создать таблицу `hints` с ключом `(operation_type_id, zone_id)`, где `zone_id = NULL`
  означает «базовые подсказки для любой зоны».
- Дополнить таблицу `zones` недостающими зонами из Excel (~16 новых зон).
- **Логика поиска** (API): сначала точное `(operation_type_id, zone_id)`,
  затем фолбэк на `(operation_type_id, NULL)`. Если ничего — пустой список.

### Минимизация данных
- **Базовые подсказки**: 62 строки с `zone_id = NULL` (лист «Уникальные задачи»)
- **ФРОВ-оверрайды**: 16 строк с `zone_id = id(ФРОВ)` (TaskID 1–14, 50, 51)
- **Итого**: ~78 строк вместо 267 в Excel

## Структура таблицы hints

```sql
hints (
  id                INTEGER PRIMARY KEY AUTOINCREMENT,
  operation_type_id INTEGER NOT NULL REFERENCES operation_types(id) ON DELETE CASCADE,
  zone_id           INTEGER REFERENCES zones(id) ON DELETE SET NULL,
  hint_1            TEXT NOT NULL,
  hint_2            TEXT,
  hint_3            TEXT,
  hint_4            TEXT,
  hint_5            TEXT,
  hint_6            TEXT,
  UNIQUE (operation_type_id, zone_id)
)
```

---

## Формат API

```
GET /references/hints?operation_type_id=8&zone_id=12
```

Ответ:
```json
{
  "ok": true,
  "data": {
    "hints": [
      "Открой в ТСД Контроль ценников и отсканируй товар",
      "Сравни цену в ТСД с ценником на полке",
      "...",
    ]
  }
}
```

Возвращает только непустые hints. Если подсказок нет — `"hints": []`.
